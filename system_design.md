## 1. 프로젝트 개요

### 1.1 목표 (재정의 경위)

| 단계 | 내용 |
|---|---|
| 원안 | 당일 개별종목 롱/숏 top5씩 자동매매, 승률 담보 |
| 1차 수정 | "당일 방향 예측 + 승률 담보"는 구조적으로 비현실적 → "비용 차감 후 양(+)의 기대값 검증"으로 목표 재정의 |
| 2차 수정 | 공매도 불가 + 소액 + 당일청산 → 개별종목 롱숏 폐기, **인덱스/섹터 ETF 방향성 베팅**으로 전환 |
| 3차 수정 | 토스증권으로 매매 확정 → **레버리지 ETF 조건부 편입**, 데이마켓으로 시간대 제약 완화 |
| 확정안 | 인덱스/섹터/레버리지 ETF의 당일 방향성 시그널 → HTML 리포트 발행 → 텔레그램 링크 발송 → **사용자 수동 실행** |

### 1.2 명시적 비목표 (Out of Scope)

- 브로커 API 연동 자동 주문 체결 — **제외** (실행 수동)
- 개별 주식 종목 선정 — **제외** (ETF만)
- 선물·옵션 — **제외** (현물 ETF만)
- 공매도 — **제외** (인버스 ETF로 대체)
- BTC 봇과의 코드/인프라 공유 — **제외** (완전 별도 프로젝트)

### 1.3 성공 / 실패(킬스위치) 기준

| 구분 | 기준 |
|---|---|
| 성공 | 2단계 게이트 통과 후, 실거래 페이즈에서 GO 합격선을 지속 충족 |

## 2. 핵심 제약 & 가정

| 제약 | 현황 | 대응 |
|---|---|---|
| 실거래 자본 $1만 미만 | 거래비용 비율이 성과를 잠식 | 초유동 ETF 위주, 비용 가정 보수적 |
| 토스증권 미국주식 수수료 0.1%/거래 | **왕복 약 0.2%** — 미국 무수수료 브로커와 다름 | 백테스트 비용 가정 왕복 0.2% + 슬리피지로 상향. GO 게이트 통과 난이도 상승을 정직하게 수용 |
| T+1 결제 | 같은 자금 하루 1회전 수준 제한 | 토스 "주식판매금 미리 받기"로 일부 완화. 1일 1사이클 전제 설계 |
| PDT 룰 | **비적용** — 토스는 한국 브로커 | 제약 해소 |
| 한국 시간대 | 토스 **데이마켓(한국시간 10:00~17:50)** 운영 → 한국 낮 매매 가능. 정규장(23:30~06:00)은 사전 지정가로 대응 | 새벽 상주 불필요. 진입·익절·손절가 사전 지정 + 데이마켓/장중 점검 |
| 레버리지 ETF 규제 | 2025년 12월부터 해외 레버리지 ETF 거래 시 **사전 의무교육 이수** 필요 (금융투자교육원, 약 1시간; 기존 레버리지 ETP 거래 경험자/국내 사전교육 이수자 면제) | 레버리지 활성화 전 사용자 교육 이수 (15절 #1) |
| 수동 실행 랙 | 시그널~체결 사이 가격 이동 | 시그널을 지정가/조건부 레벨로 산출, 백테스트에 슬리피지 반영 |

> **설계상 이점:** 진입·익절·손절 사전 지정 방식은 재량 매매보다 규칙이 명확해 **백테스트 충실도가 높습니다.**

```
[GitHub Actions — 1일 1회 배치]
   │
   ├─ 데이터 수집: yfinance(OHLCV, VIX) + FRED(매크로)
   ├─ 피처 엔지니어링: 기술적 팩터 + 매크로 레짐 변수 + (옵션)센티먼트
   ├─ 모델 추론: 저장된 가중치로 ETF별 방향 스코어[-1,+1] + 신뢰도
   ├─ 레짐 필터: 금리/DXY/크레딧/VIX → 당일 노출도 배율
   ├─ 시그널 생성: 진입가 + 익절가 + 손절가 + 적중률/손익비/기대값/팩터 기여도

                        │
                        ▼
              [사용자 수동 실행 — 토스증권]
                        │
                        ▼
              [성과 로깅: 시그널 vs 실현 — 저장소 커밋]

* 모델 학습(무거운 작업)은 Actions 외부(로컬/별도 환경)에서 수행 후 가중치 커밋.
```

## 6. 신호(팩터) 레이어 — 분야별

> 인덱스 방향성 문제에 맞게: **(a) ETF별 당일 방향 확률 + (b) ETF 간 상대강도 로테이션**.

| 분야 | 구체 신호 | 역할 | 비중 |
|---|---|---|---|
| 기술적 | 다기간 모멘텀, 단기 평균회귀, 변동성 레짐, 거래량 이상치, 시장 내부지표(advance/decline, 신고가/신저가), 추세 강도 | 주력 | 높음 |
| 매크로 | 금리·DXY·크레딧 스프레드·VIX 수준 → **레짐 필터**(노출도 배율 결정) | 노출도 조절기 | 중간 |
| 센티먼트(정치·세계정세) | 뉴스 NLP 센티먼트(3일 평균 등) | 보조 | 낮음 (Phase 2+) |

- 매크로·정치는 개별 신호로 욱여넣지 않고 "오늘 얼마나 크게 베팅할지" 상위 필터로만 사용.

## 7. 모델 레이어

| 단계 | 모델 | 목적 |
|---|---|---|
| 0. 베이스라인 | 규칙 기반(모멘텀+평균회귀 결합) | **반드시 먼저.** ML이 못 이기면 ML 불필요 |
| 1. ML 1차 | LightGBM 방향 분류/회귀 | 비선형 패턴, 빠른 반복 |
| 2. ML 2차 | GRU / learn-to-rank | 시계열 구조 + 상대강도 랭킹 |

- Qlib 활용: Alpha158/360 피처셋, TopkDropoutStrategy 응용, IC/Rank-IC 평가.
- 출력: ETF별 방향 스코어 [-1,+1] + 신뢰도(분류 확률). 신뢰도는 포지션 크기·진입 여부·**레버리지 활성화 여부** 결정에 사용.
- 검증: IC, Rank-IC, ICIR. IC가 낮으면(예: 0.01 미만) 거래비용에 신호가 묻히므로 진행 보류.
- 학습은 GitHub Actions 외부에서 수행 → 가중치를 저장소에 커밋 → Actions는 추론만.

## 8. 백테스트 프레임워크

| 항목 | 설정 |
|---|---|
| 엔진 | vectorbt(대량 탐색) + Qlib 또는 Backtrader(정밀 검증) |
| 검증 방식 | **Walk-forward** (rolling train/valid/test). 단순 1회 분할 금지 |
| **거래비용 가정(보수적)** | **수수료 왕복 0.2%**(토스증권 0.1%×2) + 스프레드·슬리피지 왕복 0.05~0.10% + 레버리지/인버스 ETF 운용보수 일할 반영 |
| 다중검정 방어 | 시도한 팩터/파라미터 조합 수 기록, deflated Sharpe 개념 적용 |
| 표본 하한 | 최소 120 거래일 이상 (Tier 1 기준) |

> **정직성 노트:** 왕복 0.2% 수수료는 미국 무수수료 브로커 가정 대비 큰 허들입니다. 당일청산 전략에서 이 비용을 이기지 못하면 전략은 폐기 대상입니다. 이 가정을 낮추지 않습니다.

## 9. 실거래 GO 게이트 (2단계)

> 사용자 선택: **공격형**(Sharpe>0.7, MDD<25%). 보정: 수익성 바는 공격형 유지, **강건성 가드 + 페이퍼 확인**으로 오버피팅 방어.

| 게이트 | 조건 | 통과 시 |
|---|---|---|
| **Tier 1** (페이퍼 진입) | 비용차감 Sharpe > 0.7 · MDD < 25% · OOS가 IS의 40%+ 유지 · walk-forward 과반 구간 양(+) · 최소 120 거래일 | 페이퍼 트레이딩 시작 |
| **Tier 2** (실거래 진입) | 3개월+ 페이퍼 실현 성과가 백테스트 대비 괴리 <40% · 페이퍼 실현 Sharpe > 0.5 | 소액 실거래 시작 (1x ETF) |
| **레버리지 활성화** | Tier 2 통과 + 페이퍼/실거래에서 **신뢰도 최상위 버킷의 실현 적중률·기대값이 입증** | 레버리지 ETF 시그널 활성화 (사이즈 축소) |

- 근거: 낮은 수익성 바는 오버피팅을 못 거른다 → 강건성 가드가 진짜 필터. 레버리지는 가장 마지막에, 입증된 버킷에서만.

## 10. 시그널 출력 스펙

각 시그널은 다음 필드를 모두 포함한다. **적중률은 단독으로 표시·해석하지 않는다.**

| 필드 | 정의 | 비고 |
|---|---|---|
| ticker / name | ETF 티커·명칭 | |
| dir | long / short(inverse) | |
| leverage | 1x / 2x / 3x | 레버리지는 활성화 조건 충족 시만 |
| entry / tp / sl | 진입가 · 익절가 · 손절가 | 지정가/조건부 레벨. 시장가 추격 금지 |
| **winrate (버킷 적중률)** | 백테스트에서 **동일 유형 신호 버킷**이 양(+) 마감한 비율 | "예측 승률" 아님 |
| sampleN / ci | 버킷 표본 수 · 신뢰구간 | 표본 작으면 신뢰 낮음 |
| payoff | 평균 익절 : 평균 손절 (손익비) | |
| expectancy | 기대값 = 적중률×평균익절 − (1−적중률)×평균손절, **비용 차감 후** | 음(−)이면 미발송 |
| confidence | 모델 신뢰도 스코어 [0,1] | 레버리지 활성화 임계치 판단 |
| factors | 발동 팩터 + 각 기여도 | 팩터 어트리뷰션 |

- **원칙:** winrate·payoff·expectancy가 **함께 양(+)**일 때만 유효 신호로 간주. expectancy가 음수면 시그널 미발송.
- HTML 리포트와 텔레그램 요약 모두 이 스펙을 따른다.

```
GitHub Actions
  └─ HTML 일일 리포트 빌드 (data 객체 주입)
       └─ GitHub Pages 배포 (https://<user>.github.io/<repo>/YYYY-MM-DD.html)
            └─ 텔레그램: 요약 텍스트 + 리포트 링크 발송
```

### 11.2 HTML 리포트 (템플릿 별도 제공: `daily-signal-report-template.html`)
- 구성: 헤더 → 레짐 스트립(레짐/노출도/시그널 수/종합 기대값) → 코어·섹터 시그널 카드 → 레버리지 시그널 카드 → 매크로 레짐 패널 → 면책 고지.
- 각 시그널 카드: 진입/익절/손절 + 적중률·손익비·기대값·표본(함께 표시) + 팩터 기여도 바 + 신뢰도.
- `REPORT` JS 객체가 **데이터 컨트랙트** — Actions 파이프라인이 이 객체를 실제 시그널로 교체.

### 11.3 텔레그램 (단방향 푸시)
- 발송: 미국장 마감 후 익일 시그널. 내용은 **요약 + HTML 리포트 링크**.
- GitHub Actions에는 상시 실행 프로세스가 없어 `/status` 등 **명령어 응답 불가** → 초기엔 단방향. 양방향 명령어는 OCI 등 상시 서버 이전 후.
- 텔레그램 메시지 예시:
  ```
  📊 [2026-XX-XX] 일일 ETF 시그널
  레짐: 중립 / 노출도 0.70× / 시그널 5개 / 종합 기대값 +0.18%
  🟢 LONG: QQQ, XLK   🔴 INVERSE: SH
  ⚡ 레버리지: TQQQ (신뢰도 0.83)
  상세(적중률·근거·팩터): https://<user>.github.io/<repo>/2026-XX-XX.html
  ⚠️ 수동 실행 · 진입가 미충족 시 미진입
  ```

## 12. 서버 / 배포 — GitHub Actions + GitHub Pages (확정)

| 구성 | 역할 | 비고 |

| 상태 저장 | 성과 로그·시그널 이력을 저장소 파일(CSV/JSON)로 커밋 | Actions는 무상태 |
| 향후 | 양방향 봇 명령어 필요 시 OCI Always Free 이전 | |

## 13. 단계별 로드맵 (주당 3~5시간 가정)

| Phase | 작업 | 산출물 | 예상 |
|---|---|---|---|
| 0 | 저장소 셋업 + 데이터 파이프라인 + 유니버스 확정 + 룩어헤드 검증 | 데이터 레이크 | 2~3주 |
| 1 | 단일 팩터 IC 검증, 무효 팩터 폐기 | 팩터 IC 리포트 | 2~3주 |

| 4 | Tier 1 게이트 → 통과 시 GitHub Actions + Pages + 텔레그램 + 페이퍼(1x) | 운영 시스템(페이퍼) | 3주 + 페이퍼 3개월 |
| 5 | Tier 2 게이트 → 통과 시 소액 실거래(1x). 이후 레버리지 버킷 입증 시 레버리지 활성화 | 실거래 + 성과 로그 | 2주~ |
| 6 | 모니터링 + 9개월 킬스위치. (선택) OCI 이전 + 양방향 봇 | 운영 체계 | 상시 |

## 부록 B. 정직성 원칙 (프로젝트 헌장)

1. 베이스라인을 못 이기는 ML은 도입하지 않는다.
2. 백테스트 성과는 비용(왕복 0.2%+) 차감 후 수치로만 보고한다.
3. 적중률을 단독으로 표시하지 않는다 — 손익비·기대값·표본과 함께만.
4. 2단계 게이트를 건너뛰지 않는다. 레버리지는 가장 마지막에 활성화한다.
5. 알파가 없다는 결론도 유효한 결과로 받아들인다.
6. 9개월 킬스위치를 감정으로 연장하지 않는다.
