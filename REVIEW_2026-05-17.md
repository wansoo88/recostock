## 1. 수정 완료 (이미 적용)

### 🔴 CRITICAL — `models/inference_v3.py:251` Threshold 불일치

**증상**:
- `score_today()` 는 `PRODUCTION_THRESHOLD = 0.58` 로 시그널 발사
- `compute_rolling_stats()` 는 `config.SIGNAL_THRESHOLD = 0.53` 로 winrate/payoff 측정
- → **리포트에 표시되는 winrate·payoff·표본수가 실제 발사 시그널과 다른 분포에서 측정된 통계**
- → 게다가 `MIN_ACTIVE_FRIDAYS = 8` 체크도 0.53 기준이라, 실제 0.58 신호 표본이 3개뿐인 종목도 통과 가능

**영향**:
- 사용자가 리포트에서 보는 WR 63% / Sharpe 1.40 같은 수치가 실제 발사 시그널의 통계가 아닐 수 있음
- honesty principle #3 위반 가능성 (적중률·표본 불일치)

**수정**: `>= PRODUCTION_THRESHOLD` 로 변경 + 설명 주석 추가.

```diff
-active_idx = common[sig.reindex(common) >= config.SIGNAL_THRESHOLD]
+active_idx = common[sig.reindex(common) >= PRODUCTION_THRESHOLD]
```

**Note**: 학습된 v3 모델은 그대로 유효. 단, 다음 페이퍼 백테스트에서 rolling stats가 진짜 분포로 측정될 예정 → 이전 리포트 수치와 약간 달라질 수 있음.

### 🟠 HIGH — `models/inference_v3.py:202` Series.get() NaN propagation

**증상**:
- `ema = float(ema_proba.get(ticker, raw))`
- `Series.get(key, default)` 는 **key가 존재하지만 값이 NaN인 경우 NaN을 반환** (default는 missing key에서만 작동)
- `float(NaN)` 은 성공 → NaN이 `Signal.is_valid()` 로 전파 → NaN 비교는 항상 False → 시그널이 조용히 누락됨

**시나리오**: EMA 워밍업 부족, 신규 ETF 추가 직후, 데이터 누락 등에서 발생 가능

**수정**:
```diff
-ema = float(ema_proba.get(ticker, raw))
+ema_raw = ema_proba.get(ticker, raw)
+ema = float(raw) if pd.isna(ema_raw) else float(ema_raw)
```

# 6. WR 70% 목표 분석 (부재 중 추가 조사)

사용자 요청: "WR 70%까지 도달할 수 있도록 노력". 정직한 결론을 먼저 보고합니다.

## 6.1 가장 중요한 발견: **알파 decay**

실제 페이퍼 거래 270건의 연도별 WR:

| 연도 | WR | 표본 | 평균 일수익 |
|------|-----|------|-------------|
| 2020 | **68.2%** | 22 | +2.32% |
| 2021 | **69.1%** | 68 | +2.12% |
| 2022 | 43.3% | 30 | -2.14% |
| 2023 | 61.4% | 44 | +0.92% |
| 2024 | 56.6% | 53 | +0.84% |
| 2025 | 46.0% | 37 | +0.35% |
| **2026 YTD** | **18.8%** | 16 | -0.47% |
| **2026 Q2** | **0.0%** | 6 | -0.36% |

**해석**:
- 시스템은 **2020-2021에 이미 WR 69% 달성** — 70% 목표는 데이터상 가능
- 그러나 **2022년부터 명백히 decay** 시작
- **2026 Q2의 WR 0%는 honesty principle #6 (9개월 킬스위치) 영역**
- 표면적 OOS 백테스트(WR 57%)와 실제 페이퍼(WR 19%)의 괴리가 vipeline이 아닌 alpha decay 신호

## 6.2 우선 fix가 필요한 production bug — 페이퍼 트래커 중복

`paper/tracker.py` 분석 결과 다음 데이터 발견:
- **2026-05-15 DIA 5건 동일 거래** (1 + 중복 4) — 모두 entry=exit=500.80, pnl=-0.25% (비용만)
- **2026-05-08 SPY** — 같은 날 open/close, pnl=-0.25%
- 총 same-day open/close 거래 5건

**원인 추정**: `run_daily.py`가 같은 Friday에 여러 번 실행됨. ema_proba가 threshold 0.58 근처(0.5954, 0.5741)일 때 작은 데이터 변경으로 시그널이 on→off→on을 반복 → 매번 open+close. 비용만 까먹음.

**영향**: 최근 페이퍼 WR이 실제보다 더 나쁘게 보임 (가짜 -0.25% 거래들이 분모에 들어감). 진짜 alpha decay와 트래커 버그가 섞여있음.

**제안 fix**:
1. `paper/tracker.py` — 같은 날 open & close 방지: open 후 최소 1거래일 보유 강제
2. GitHub Actions `concurrency` 설정 점검 — `daily-signal` 그룹이 진짜 작동하는지
3. 5월 중복 DIA 4건 + same-day exit 5건 제거 후 페이퍼 통계 재계산 → 진짜 최근 WR 측정

## 6.3 이미 시도된 WR-pushing 기법 (모두 한계 도달)

`scripts/push_wr_higher.py`에서 이미 측정한 결과:

| 전략 | 최고 WR (n≥10, 12m) |
|------|-------------------|
| Threshold sweep (0.53~0.62) | **57.1%** @ 0.58 |
| Top-K (3, 5, 7) | 57.1% @ K=3 또는 5 |

→ **conventional WR-pushing 기법은 전부 exhausted**. 0.58은 이미 sweet spot.

## 8.6 WR 70%에 대한 정직한 결론 (재방문)

**시도한 것 (모두 측정 완료)**:
1. ✅ Threshold sweep 0.55~0.75
2. ✅ Top-K sweep K=1~7
3. ✅ EMA vs Raw proba 비교
4. ✅ Stop-loss 1.5%~3.0%
5. ✅ Take-profit 조합
6. ✅ Fractional allocation (100%/50%/30%/20%)
7. ✅ Walk-forward holdout 분리

**측정된 사실**:
- **Holdout에서 WR 70% 달성 가능한 안정 전략 없음** (n>=30 기준)
- WR 70% 근접하는 구간들이 존재 (2023 상위 10% WR 75.5%, 2025Q4 82.6%) — 특정 regime에서만
- 더 strict하게 필터링할수록 거래수 부족 → 통계 신뢰도 하락
- **그러나** WR 48-59% + 적절한 SL/payoff로 **수익성 있는 전략 발견** ← 이게 진짜 알파

**WR 70%는 잘못 정의된 목표**:
- WR 70% + payoff 0.4 = 손실 (요구되는 payoff 0.43 미달)
- WR 50% + payoff 1.4 = 수익
- → **Expectancy(E%)가 진짜 지표**. WR 단독은 misleading
