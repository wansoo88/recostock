> 사용자 요청("토스증권 API 통한 자동 거래도 한번 고려해보고")에 대한 조사·권고.
> 조사 방법: 병렬 웹 검색 → 공식 문서 원문 확인 → 핵심 주장 7건 재검증. 출처 병기.

## 1. 조사 결과 요약

### 토스증권 — 공개 Open API **존재** (2026-05~06 신규 출시, 순차 오픈 중) [확실도 높음]

- 주문 생성·정정·취소 REST 엔드포인트 공식 문서화: `POST /api/v1/orders`,
  `/api/v1/orders/{orderId}/modify`, `/api/v1/orders/{orderId}/cancel`.
  인증 OAuth 2.0(Client Credentials) — **WTS(PC웹) 설정 > Open API에서 개인이 직접 키 발급**.
  - https://openapi.tossinvest.com/openapi-docs/overview.md · https://developers.tossinvest.com/docs
- **미국 주식 지원 명시**: "국내(KRX) 및 미국 주식의 시세, 종목정보, 환율, 계좌, 보유주식, 주문 기능 제공".
- 타임라인: 5월 중순 사전신청(2주에 ~5.5만 명) → 6월 초부터 순차 오픈 → **2026 H2 완성 목표**.
  - 이데일리 2026-06-11: https://marketin.edaily.co.kr/News/ReadE?newsId=04795366645480736
- 제약: 현재 **REST 전용**(WebSocket 미제공 → 체결 확인은 폴링), rate limit(ORDER 초당 6회 등),
  롤아웃 초기라 안정성 유동적. 약관 원문(자동매매 명시 허용 여부)은 JS 렌더링으로 미확인 —
  신청 시 확인 필요: https://home.tossinvest.com/ko/terms/v2?id=752
- 주의: 앱 내 "예약주문/자동주문"(2021~)은 이번 공개 API와 별개 기능.

### 차선책 — 한국투자증권 KIS Developers [높음]

- 해외주식 주문 API 공식 지원(`/uapi/overseas-stock/v1/trading/order`, 매수 TR TTTT1002U).
  **모의투자 지원**(페이퍼 검증과 결 맞음). 미국 정규장 **매수는 지정가/LOO/LOC만**(시장가 없음)
  → 본 시스템의 종가 기반 진입과 호환. **소수점 주문 API 없음**(미니스탁 앱 전용).
  - https://github.com/koreainvestment/open-trading-api
- 기타: LS증권·DB증권도 미국주식 주문 API 제공. 키움은 국내 전용(해외주식 API 없음).

### SPXL(레버리지 ETP) 선행 요건 [높음]

- **2025-12-15부터** 해외 레버리지 ETP 신규 매수 시 **사전교육 이수번호 등록 의무**
  (금융투자교육원 1시간 과정 — 기존 투자자/기존 이수자 면제).
- **2026-05-22부터** 해외 레버리지 '최초' 거래 계좌에 **기본예탁금 1,000만원**(기존 경험자 제외).
  - 금융위: https://www.korea.kr/news/policyNewsView.do?newsId=148963133
- 계좌 단위 요건이므로 채널(앱/API) 불문 적용으로 보임[추정]. 본 계좌가 이미 SPXL을 거래 중이면
  대체로 충족 상태일 것 — 신청 시 확인.

### 법적/약관 [높음]

- **본인 계좌·본인 자금** 자동매매는 합법(인허가 불요). 증권사 약관도 자동주문 프로그램을 명시 허용(KIS 등).
- 경계 2가지: ① 허수성 호가 반복 알고리즘은 개인도 시세조종 제재 대상(시타델 과징금 118.8억 사례),
  ② **타인 자금 수탁 운용은 미등록 투자일임업**(형사처벌). 본 시스템(1일 1회 종가 리밸런스, 본인 자금)은
  ①·② 모두 해당 없음.

## 2. 프로젝트 게이트와의 관계 (부록 B 준수)

- 자동화는 **실행 채널**의 개선일 뿐, **실자본 전환 결정**과 분리되지 않는다.
  Tier-2(페이퍼 3개월 ~2026-08-29 + Sharpe>0.5 + 괴리<40%) **통과 전 실자본 자동주문 금지**
  — "2단계 게이트를 건너뛰지 않는다"(원칙 4).
- 자동화가 더하는 가치는 알파가 아니라 **실행 비용·실수 감소**: 누락 없는 리밸런스, 진입가 일관성,
  손절선 이탈 다음 날 기계적 청산. 기대수익을 바꾸지 않으므로 백테스트·게이트 수치에 영향 없음.

## 3. 권고 로드맵

| 단계 | 시기 | 내용 | 리스크 |
|------|------|------|--------|
| **0. 사전신청** | 지금 | WTS > 설정 > Open API에서 신청·키 발급(순차 승인 대기열). 약관의 자동주문 조항 확인 | 없음(무료) |
| **1. 읽기 전용 통합** | 검증 기간 중 | 잔고/체결 조회 API로 **실계좌 보유 vs 목표 블렌드 자동 대조** — decision의 '현재 보유'를 트래커 기록 대신 실잔고로 정밀화. 주문은 계속 수동 | 낮음 (조회만) |
| **2. 반자동** | 게이트 통과 후 | 파이프라인이 **주문 초안 생성**(지정가) → 사용자 1탭 승인 → 발사. 드라이런 모드 기본 | 중간 |
| **3. 완전 자동** | 반자동 1개월+ 무사고 후 | 무인 리밸런스. 아래 안전장치 전제 | 높음 |

**완전 자동의 필수 안전장치** (설계 시점에 전부 구현):
킬스위치(env 1개로 전면 중지) · 일일 주문 횟수/금액 상한 · 지정가만(±밴드) · 멱등 키로 중복 발사 방지 ·
주문 전 잔고·미체결 검증 · stale-data 가드 연동(지연 시 발사 금지) · 모든 주문 텔레그램 즉시 통보 ·
주문 실행은 GitHub Actions가 아닌 **우분투 서버**에서(거래 권한 키를 GH secrets에 두지 않음).

## 4. 미해결/확인 필요

- 토스 Open API 약관 원문의 자동주문 조항(신청 후 확인).
- 소수점 주문 API 지원 여부(문서에 명시 없음 — 미지원이면 정수주 반올림 정책 필요, 리포트 계산기와 정합).
- 환전 API 유무(원화→달러), BIL/SGOV 주문 가능 여부(미국 ETF이므로 가능 추정).
- 순차 오픈 승인 소요 기간.

## 5. 결론

**지금 액션은 두 가지뿐**: ① 토스 Open API 사전신청(대기열), ② 승인되면 읽기 전용(잔고 조회)부터 연동.
주문 자동화 코드는 **게이트 통과가 가시화되는 8월에 착수**해도 늦지 않다 — 그때 이 문서의 안전장치
목록이 설계 입력이 된다. 검증 중인 전략 자체는 어떤 변경도 없다.
